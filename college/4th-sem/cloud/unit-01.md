# Unit 1 — Introduction to Cloud Computing

## Topic 1: From Collaborative to the Cloud (Very Detailed Notes)

---

# 1. From Collaborative to the Cloud — Introduction

**From Collaborative to the Cloud** means how computing evolved from **people working together by sharing resources** to **modern cloud computing where everything is shared over the internet.**

Earlier:

- Users worked individually
    
- Data stored locally
    
- Limited sharing
    

Then:

- Collaborative systems allowed **multiple users to work together**
    

Finally:

- Cloud computing provides **global collaboration + shared resources + on-demand services**
    

So this topic explains **evolution toward Cloud Computing**

---

# 2. What is Collaborative Computing?

Collaborative computing means:

> Multiple users working together using shared computing resources to achieve a common goal.

Examples:

- Multiple users editing same document
    
- Team accessing shared database
    
- Group working on same software project
    

Features:

- Resource sharing
    
- Communication between users
    
- Group work support
    
- Remote access
    

---

# 3. Problems with Traditional Collaborative Computing

Earlier collaborative systems had many limitations:

### 1. Limited Storage

- Data stored on local servers
    
- Storage capacity limited
    

### 2. High Cost

- Organizations had to buy hardware
    
- Maintenance cost high
    

### 3. Limited Accessibility

- Access only inside organization network
    
- No global access
    

### 4. Scalability Problem

- Adding new users difficult
    
- System crashed under heavy load
    

### 5. Maintenance Issues

- Manual updates required
    
- Server downtime
    

These problems led to **Cloud Computing**

---

# 4. Evolution: Collaborative → Cloud Computing

Evolution path:

Collaborative Computing  
↓  
Distributed Computing  
↓  
Grid Computing  
↓  
Virtualization  
↓  
Cloud Computing

This shows cloud computing is **advanced form of collaborative computing**

---

# 5. How Cloud Solves Collaborative Computing Problems

|Problem|Cloud Solution|
|---|---|
|Limited storage|Unlimited cloud storage|
|High cost|Pay as you use|
|Limited access|Access from anywhere|
|Scalability issue|Auto scaling|
|Maintenance|Cloud provider handles|

---

# 12. Short Note (5 Marks Ready)

From Collaborative to the Cloud describes the transition from traditional collaborative computing to cloud computing. In collaborative computing, users share resources within a limited environment. These systems suffered from limited storage, high cost, and poor scalability. Cloud computing solves these problems by providing shared resources, on-demand services, remote accessibility, and scalability over the internet. Examples include Google Docs and cloud-based project tools. Cloud computing enhances collaboration by allowing multiple users to work simultaneously from different locations.

# Unit 1 — Topic 2

# A Short History of Client–Server Computing (Detailed Notes)

---

# 1. Introduction

Client–Server computing is a model where:

- **Client** requests services
    
- **Server** provides services
    

This model replaced **centralized mainframe computing** and became the **foundation of Cloud Computing**.

So this topic explains **how computing evolved before cloud**

---

# 2. What is Client?

Client is:

> A computer or software that requests services from server.

Examples:

- Web browser
    
- Mobile app
    
- Desktop application
    
- ATM machine
    

Client sends **request**

---

# 3. What is Server?

Server is:

> A powerful computer that provides services to clients.

Server responsibilities:

- Store data
    
- Process request
    
- Send response
    
- Manage users
    

Examples:

- Web server
    
- Database server
    
- File server
    
- Mail server
    

---

# 4. Client–Server Architecture

Working:

Client → Request → Server  
Server → Response → Client

Example:  
You open website

Browser (Client) → request → Web server  
Server → sends webpage → Browser

---

# 5. History of Client–Server Computing

## Phase 1: Mainframe Computing (1960s–1970s)

- One powerful central computer
    
- Many dumb terminals connected
    
- Users cannot process locally
    
- Everything controlled by mainframe
    

Problems:

- Expensive
    
- Single point failure
    
- No flexibility
    

---

## Phase 2: Personal Computer Era (1980s)

- Personal computers introduced
    
- Users process locally
    
- No sharing between computers
    

Problems:

- Data duplication
    
- No centralized control
    
- Difficult collaboration
    

---

## Phase 3: Client–Server Computing (1990s)

Solution introduced:

- Clients handle user interface
    
- Server handles data + processing
    
- Network connects them
    

This became **Client–Server Model**

---

# 6. Types of Client–Server Architecture

## 1. Two Tier Architecture

Client ↔ Server

Client:

- UI
    
- Logic
    

Server:

- Database
    

Example:  
Desktop app + database server

---

## 2. Three Tier Architecture

Client → Application Server → Database Server

Tier 1:  
Client (UI)

Tier 2:  
Application server (logic)

Tier 3:  
Database server (data)

Example:  
Web application

Browser → Web server → Database

---

# 7. Characteristics of Client–Server Computing

### 1. Centralized Data

Data stored on server

### 2. Multiple Clients

Many clients connect to server

### 3. Request Response Model

Client sends request

### 4. Resource Sharing

Server resources shared

### 5. Network Based

Requires network

---

# 8. Example (Important for Exam)

Example: Banking System

Client:  
ATM machine

Server:  
Bank database

Process:  
ATM → request balance  
Server → sends balance

This is Client–Server computing

---

# 9. Advantages of Client–Server Computing

### 1. Centralized Control

Easy management

### 2. Data Security

Server controls access

### 3. Resource Sharing

Many users share server

### 4. Easy Backup

Backup only server

### 5. Scalable

Add more clients

---

# 10. Disadvantages

### 1. Server Failure

Whole system stops

### 2. Network Dependency

Needs network

### 3. High Server Cost

Powerful server required

### 4. Performance Bottleneck

Too many clients slow server

---

# 11. Client–Server vs Cloud (Important)

Client–Server:

- Limited scalability
    
- Single server
    
- Manual management
    

Cloud:

- Auto scaling
    
- Multiple servers
    
- Fully managed
    

Cloud is **advanced form of client–server**

---

# 12. Exam Definition (Write This)

Client–Server computing is a distributed computing model in which a client sends requests for services and a server processes the requests and returns the response over a network.

---

# 13. 5 Marks Answer Ready

Client–Server computing is a computing model where clients request services from centralized servers. This model replaced mainframe computing and allowed multiple users to share resources. Clients handle user interface while servers handle data storage and processing. Examples include web applications and banking systems. It offers centralized control, security, and resource sharing but suffers from server dependency and scalability issues. This model later evolved into cloud computing.
